# BubblePulse — 原型设计文档 (DESIGN.md)

> 设计目标：让品牌内容团队打开产品后，看到的数据天然是"跟我的品牌受众最相关的"，不需手动筛选。
> 匹配层展示确定性计算结果，AI 只在 `/brief` 统一介入。

---

## 一、设计系统

### 视觉调性

| 维度 | 选择 | 理由 |
|------|------|------|
| 风格方向 | Clean Data-Dense Dashboard | 数据咨询公司产品，专业但不过于冷感 |
| 配色 | 深蓝 + 琥珀色高亮 | 深蓝 = 数据信任感，琥珀 = 热度/行动召唤 |
| 字体 | Inter（UI）+ 系统默认中文 | 现代SaaS标配，数字可读性好 |
| 圆角 | 8px（卡片）/ 6px（按钮） | 专业感，不过于圆润 |
| 图标 | Lucide React | shadcn/ui 原生配套 |

### 配色方案

```
Background:  #0F172A (slate-950) — 深色模式主背景
Sidebar:     #1E293B (slate-800) — 侧边栏
Card:        #1E293B (slate-800) — 卡片背景
Card Hover:  #334155 (slate-700) — 卡片悬浮
Text:        #F8FAFC (slate-50)  — 主文字
Text Muted:  #94A3B8 (slate-400) — 次级文字
Accent:      #F59E0B (amber-500) — 热度/行动高亮
Success:     #10B981 (emerald-500) — 上升/机会
Danger:      #EF4444 (red-500)    — 下降/风险/警告
Border:      #334155 (slate-700) — 分割线
```

### 组件模式

| 模式 | 用途 | shadcn 组件 |
|------|------|------------|
| 数据卡片 | 热点/KOL/IP 的展示单元 | `Card` + 自定义内容 |
| 筛选栏 | 平台/人群/格式的过滤 + 品牌预设开关 | `Select` + `Badge` + `Toggle` |
| 趋势指标 | 热度涨跌、匹配度评分、生命周期标记 | 自定义 `TrendBadge`、`LifecycleBadge` |
| 简报内容 | AI生成的周报文档 | `Card` + amber 左边框 + 结构化布局 |
| 日历 | 营销节点展示 | 自定义 `Calendar` 组件 |
| 数据可视化 | 趋势曲线、热点对比 | `Recharts` |

## 五、交互规范

### 颜色语义

| 颜色 | 含义 | 使用场景 |
|------|------|---------|
| 🟢 emerald | 机会、高匹配、上升 | 受众重合>90%、rising、高可行性 |
| 🟡 amber | 注意、中等、需判断 | 受众重合80-89%、peak、中可行性 |
| 🔵 blue | 信息、可关注 | 受众重合70-79%、圈层热点 |
| 🔴 red | 警告、风险、下降 | 风险标记、declining、低可行性 |
| ⚪ slate | 中性 | 受众重合<70% |

### 品牌预设模式

- **开启**：所有筛选项自动填入品牌设置中的TA参数，筛选栏显示"🔗 品牌预设"标识
- **关闭**：自由筛选，所有筛选项可手动修改
- 切换不刷新页面，列表即时更新

### 空状态

| 场景 | 展示 |
|------|------|
| 未设置品牌画像 | "设定你的品牌画像，解锁个性化匹配" + CTA到 `/brand/settings` |
| 无匹配热点 | "本周暂无高重合度热点。以下是全平台热门内容" |
| 无匹配KOL | "调整筛选条件，或扩大粉丝量范围" |
| 无IP联动机会 | "暂无符合条件的IP，扩大类型范围获取更多机会" |

### 加载状态

- 页面切换：skeleton卡片
- 筛选变更：列表区域刷新 + 结果计数更新
- AI简报生成：渐进式加载动画（2-3秒）
