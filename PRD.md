# BubblePulse — 产品设计文档 (PRD)

> 面向出海餐饮品牌的海外内容策略操作系统。
> 不是一份"热门榜单"，而是帮助品牌在海外文化语境下建立本土相关性、信任感与增长的生存系统。
> 把战术性的追热点，进化成战略性的本地化内容敏捷力。

---

## 一、产品定位

### 核心洞察

茶饮品牌出海最大的壁垒不是产品，是**文化陌生感**。热点内容是当地用户在某一时刻的集体情绪公约——
掌握热点等于拿到了当地文化的实时词典，让品牌用当地人的语言说话。

BubblePulse 解决的是一个具体的问题：
**从海量海外平台内容中，找到"与我的品牌受众最相关"的热点，并转化为可执行的内容计划。**

核心差异：**我们按受众重合度排序，不按品类相关性排序。**
一个 22 岁的 CHAGEE 消费者也喜欢 Genshin Impact 和 K-pop——这些内容跟茶饮无关，但跟你的消费者有关。

### 案例品牌

Demo 以 **CHAGEE（霸王茶姬）** 为例：
- 定位：新中式茶饮 / Premium
- 调性：Oriental Aesthetic · Modern · Minimalist
- 目标市场：美国、英国、东南亚
- 核心TA：22-35F 新中产女性（东方美学 / wellness / fashion / travel）
- 第二TA：18-28 文化认同年轻群（anime / K-pop / Genshin Impact / Asian culture）

### 为谁解决什么问题

| 角色 | 痛点 | BubblePulse 交付 |
|------|------|-------------------|
| 品牌内容负责人 | "每周都要想拍什么，不知道当地什么会火" | 受众匹配的热点 + 生命周期判断 + 内容方向 |
| 市场营销/增长 | "投放ROI低，不知道该找谁合作" | KOL匹配推荐，按受众重合度和ROI排序 |
| 联名/IP合作 | "不知道当下什么IP值得联动" | IP热度追踪 + 可行性评估 + 竞品占用情况 |

## 二、需求覆盖框架

产品的设计依据来自茶饮品牌出海对"海外平台热点"的四层需求：

```
第一层：监测阵地与信号类型
  → 跨平台数据聚合 + 内容形式分类 + 圈层文化热点
第二层：筛选与评估
  → 受众重合度匹配 + 生命周期判断 + 风险审查
第三层：热点-品牌连接
  → 内容方向建议 / 产品启发 / KOL匹配
第四层：可持续能力
  → 营销日历（节点型热点预判）
```

### 需求-功能映射

| 需求层 | 需求 | 产品功能 | 优先级 |
|--------|------|---------|--------|
| 第一层 | 跨平台热点聚合 | `/trends` 趋势引擎（TikTok / Instagram / YouTube Shorts / YouTube） | P0 |
| 第一层 | 内容形式分类 | hashtag / audio / challenge / short_video / long_video | P0 |
| 第一层 | 圈层文化热点 | `/ip-tracker` IP联动追踪（独立模块，决策周期和逻辑不同于短视频热点） | P0 |
| 第一层 | 音频/视觉趋势 | 后续版本加入 | 路线图 |
| 第二层 | 受众重合度匹配 | 基于品牌TA画像 × 热点受众画像的确定性计算（非AI） | P0 |
| 第二层 | 品牌预设模式 | 筛选栏一键填入品牌设置中的TA参数 | P0 |
| 第二层 | 生命周期判断 | 涨速+热度基数+跨平台验证 → 窗口期预估 | P0 |
| 第二层 | 风险审查 | 文化敏感/宗教/品牌安全标记 | P1 |
| 第三层 | 内容方向建议 | 由 `/brief` AI简报统一输出（各页面不做分散的AI建议） | P0 |
| 第三层 | KOL匹配 | `/kol` 受众重合度 + 费用预估 + 合作历史 | P0 |
| 第四层 | 营销日历 | `/calendar` 按地区标注重要文化节点 | P1 |

### 有意不做或暂缓的

| 内容 | 原因 |
|------|------|
| 竞品搜索/监控 | 平台API不支持按账号名获取竞品内容详情，demo真实性风险高 |
| 话术层转化建议 | 首版聚焦内容方向，话术层后续迭代 |
| 本地信号人工输入 | 第四层能力，远期路线图 |
| 热启动审批流程 | B2B工作流，不在demo范围 |
| 复盘知识库 | 需要真实运营数据积累，demo无法演示 |

## 三、核心模块

### 模块 1：实时热点引擎（`/trends`）

**按受众重合度排序，不是品类相关性。匹配是确定性计算，不是AI猜测。**

每条热点回答三个问题：
1. 我的受众在消费这个内容吗？（受众重合度）
2. 还有多久的窗口期？（生命周期判断——涨速/基数/跨平台验证）
3. 我该怎么做？（去 `/brief` 生成简报获得统一AI建议）

| 字段 | 说明 |
|------|------|
| 热点名称 | 跨平台标准化后的话题/格式/audio名 |
| 平台 | TikTok / Instagram / YouTube Shorts / YouTube |
| 热度指数 | 标准化 0-100 + 涨跌趋势 |
| 受众重合度 | 该热点的受众画像与品牌TA的重合百分比（主导排序） |
| 生命周期 | rising / peak / declining + 预估窗口期 |
| 跨平台标记 | 单平台 / 跨平台共振 |
| 竞品密度 | 同类品牌是否已跟进（影响"追不追"的判断） |
| 风险标记 | 可选——文化敏感/宗教/品牌安全（仅标记，不拦截） |

**筛选维度**：平台、市场、人群（年龄/性别/兴趣）、内容格式、热度阶段、匹配度阈值
**品牌预设模式**：一键开关，开启后所有筛选项自动填入品牌设置中的TA参数

### 模块 2：KOL 发现与匹配（`/kol`）

**按受众重合度和ROI排序，不是粉丝数。**

| 字段 | 说明 |
|------|------|
| 创作者信息 | 平台 / handle / 粉丝量 / 平均互动率 |
| 内容风格标签 | ASMR / aesthetic / comedic / educational / lifestyle / food_review |
| 受众重合度 | KOL粉丝与品牌TA的重合百分比 |
| 品牌匹配分 | 综合评分 0-10 |
| 近期爆款 | TOP3 高互动内容 |
| 合作费用预估 | 基于粉丝量和互动率的区间估算 |
| 合作历史 | 过往品牌合作记录 |

**筛选维度**：粉丝量区间、内容风格、受众重合度、平台、地区

### 模块 3：IP 联动追踪（`/ip-tracker`）

**独立于趋势引擎。IP热度的生命周期（3-12个月）和联动决策逻辑（授权费+产品开发+campaign）与短视频热点（3-14天）完全不同。IP数据同样按受众重合度筛选。**

| 字段 | 说明 |
|------|------|
| IP名称/来源 | 动漫 / 游戏 / 影视 / 角色 / meme |
| 当前热度 | 0-100 + 趋势方向 |
| 受众重合度 | IP粉丝与品牌TA的重合百分比 |
| 受众画像 | 核心粉丝人群描述 |
| 联动先例 | 过往品牌合作案例 |
| 联动可行性 | 高 / 中 / 低 |
| 竞品占用 | 是否已被同类品牌占用 |

颜色编码：
- 🟢 高可行性 + 未被占用 = 机会窗口
- 🟡 中可行性 / 拥挤但有机会 = 需差异化方案
- 🔴 低可行性 / 已被独占 = 暂时观望

### 模块 4：AI 策略简报生成（`/brief`）

**产品中唯一的AI交互入口。** 匹配层使用确定性算法（受众画像 × 热点画像 = 重合度），
AI只在最后一步介入——接收已排序的热点/KOL/IP数据，按简报模板规则产出结构化输出。

```
周度简报结构：
├── 本周TOP内容机会（按匹配度排序 + 窗口期 + 行动建议）
├── 推荐合作KOL（Top 3 + 受众重合度 + 费用）
├── 值得关注的IP（1-2个 + 时间窗口）
└── 下周内容日历建议
```

输出形式：页面内一键生成，支持导出飞书 / Notion / PDF。

### 模块 5：营销日历（`/calendar`）

**按地区标注可预测的重要文化节点——这些是确定性热点，不需要抓取，只需要提前规划。**

| 地区 | 节点示例 |
|------|---------|
| 美国 | Super Bowl (2月) / Coachella (4月) / 感恩节 (11月) / 圣诞 (12月) |
| 英国 | 圣诞 / Boxing Day (12月) / Wimbledon (7月) / Notting Hill Carnival (8月) |
| 东南亚 | 斋月 (日期浮动) / 泼水节 (4月) / 排灯节 (10-11月) / 农历新年 |
| 澳洲 | Australia Day (1月) / Vivid Sydney (5-6月) / 圣诞 (12月) |

每条标注：日期 / 地区 / 节点类型 / 茶饮关联建议。

### 模块 6：品牌画像设置（`/brand/settings`）

**产品的逻辑起点。** 品牌画像是所有"匹配度"计算的锚点——
它证明了 BubblePulse 是从品牌自身出发找交叉热点，而非通用热门榜单。

| 设置项 | 说明 |
|--------|------|
| 基本信息 | 品牌名、品类、价格带、视觉风格 |
| 目标市场 | 多选：美国/英国/澳洲/东南亚 |
| 目标人群 | 年龄范围、性别、兴趣标签（可多组） |
| 视觉调性 | 品牌的审美风格标签 |

## 四、数据模型

```
Brand（品牌画像）
├── name, category, price_tier
├── visual_style: [string]
├── markets: [string]
└── target_audiences: [{ age_range, gender, interests[] }]

ContentItem（热点/KOL内容/竞品内容统一模型）
├── id, platform, type
├── title, description
├── metrics: { views, likes, shares, comments, growth_rate }
├── format: hashtag / audio / challenge / short_video / long_video
├── tags: [string]
├── demographic_affinity: { age_18_24, age_25_34, age_35_44, female, male }
├── audience_overlap: float          ← 主导匹配指标（确定性计算）
├── lifecycle: {
│     stage: rising / peak / declining
│     estimated_window: string       ← "3-7天" / "1-2周" / "不建议进入"
│     cross_platform: boolean        ← 是否跨平台共振
│     competitor_density: low/medium/high
│   }
├── risk_flags?: [{                  ← 可选
│     type: cultural_sensitivity / religious / brand_safety
│     level: low / medium / high
│     note: string
│   }]
└── created_at

KOL（创作者）
├── handle, platform, display_name
├── followers, avg_engagement_rate
├── content_style_tags: [string]
├── audience_profile: { age, gender, interests, region }
├── audience_overlap: float
├── recent_viral_posts: [ContentItem]
├── brand_collab_history: [{ brand, date, type }]
├── estimated_cost_range: { min, max }
└── brand_fit_score: 0-10

IP（联动资产）
├── name, category
├── heat_score, trend_direction
├── audience_overlap: float
├── audience_profile: string
├── collab_precedents: [{ brand, year, description, social_impression }]
├── feasibility: high / medium / low
└── competitor_occupied: boolean
```

## 七、AI 使用边界

```
确定性计算（不用AI）                  AI（仅 /brief）
───────────────────                  ──────────────
受众画像 × 热点画像 = 重合度          收到已排序的数据
热度涨速 + 基数 + 跨平台 = 生命周期    收到品牌画像
IP可行性 + 竞品占用 = 机会评估        按简报模板规则
KOL受众重合 × 互动率 = 匹配分        产出结构化周报
                                     包含行动建议
```

匹配层是数学，用户可以追溯"为什么这个热点排在这里"。
生成层是AI，用户主动触发，只在需要组织成计划时使用。

## 八、技术方案

| 层 | 选型 | 理由 |
|----|------|------|
| 框架 | Next.js 14 (App Router) | React生态，API routes可mock后端 |

- 50-80条热点（跨 TikTok / Instagram / YouTube Shorts / YouTube）
- 20-30个KOL + 15-25个IP

- 1-2条带风险标记示例
- Demo品牌：CHAGEE
